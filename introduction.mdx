HASH is a self-building, open-source database that grows, structures, and checks itself. It integrates data in near-realtime and provides a powerful set of interfaces so that information can be understood and used in any context.

Intelligent, autonomous agents can be deployed to grow, check, and maintain the database — integrating and structuring information from the public internet as well as your own connected private sources. Users, including those who are non-technical, can visually browse and manage both entities (data) and types (schemas). HASH acts as a source of truth for critical data, no matter its source, and provides a platform for high-trust, safety-assured decision-making.

<Note>
  HASH is available as a hosted service at [app.hash.ai](https://app.hash.ai) or you can run it yourself. Self-hosting is experimental and not yet officially supported — see [Self-Hosting HASH](/self-hosting) for details.
</Note>

## Architecture

HASH is composed of several services that work together:

**Frontend** — A Next.js web application served at `localhost:3000` in local development. It provides the visual interface for browsing entities, defining types, running flows, and managing your workspace.

**Graph API** — A high-performance Rust service that acts as the query layer over the main datastore. It exposes a REST API for creating, querying, and managing entities, types, and their relationships. The Graph service uses PostgreSQL for persistent storage and includes a strongly-typed ontology with full temporal versioning.

**AI worker** — A TypeScript-based [Temporal](https://temporal.io) worker (`hash-ai-worker-ts`) that executes AI-powered workflows. It connects to OpenAI and Anthropic APIs to power research flows, question-answering, and entity enrichment tasks.

**Integration worker** — Manages two-way synchronization of external data sources with your HASH knowledge graph.

**Browser plugin** — A browser extension that lets you capture structured entities from any webpage and push them directly into HASH.

**External services** — HASH depends on Postgres (data storage), Ory Kratos (authentication), Redis (queuing), and Temporal (workflow orchestration). These run as Docker containers managed by the `yarn external-services` commands.
