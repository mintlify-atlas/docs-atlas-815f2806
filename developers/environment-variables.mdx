---
title: "Environment Variables"
description: "Reference for environment variables that developers need to configure when running HASH locally or self-hosting."
---

HASH is configured through environment variables. The default values defined in `.env` and `.env.development` are sufficient to run the application in development mode ŌĆö you do not need to set any variables unless you want to enable optional features such as AI, email, or cloud storage.

## Configuration file precedence

Variables are loaded in the following order (later files take precedence):

1. `.env` ŌĆö committed repository defaults
2. `.env.development` ŌĆö development-specific overrides (committed)
3. `.env.local` ŌĆö your local secrets and personal overrides (**not committed**)

<Warning>
  Never commit secrets to `.env` or `.env.development`. Put all secrets and values that must remain private in `.env.local`.
</Warning>

<Tip>
  For a minimal AI-capable local setup, only the variables in `.env.local` below are required. Everything else has working defaults.

  ```bash
  OPENAI_API_KEY=your-openai-api-key
  ANTHROPIC_API_KEY=your-anthropic-api-key
  HASH_TEMPORAL_WORKER_AI_AWS_ACCESS_KEY_ID=your-aws-key-id
  HASH_TEMPORAL_WORKER_AI_AWS_SECRET_ACCESS_KEY=your-aws-secret
  ```
</Tip>

  <Accordion title="API server">
    General configuration for the `hash-api` Node.js server.

    | Variable | Default | Description |
    |----------|---------|-------------|
    | `NODE_ENV` | `development` | Runtime environment. Controls default log levels and output formatting. Set to `production` for production deployments. |
    | `PORT` | `5001` | Port the API server listens on. |
    | `API_ORIGIN` | `http://localhost:5001` | The origin that the API service can be reached on. Used for generating absolute URLs. |
    | `SESSION_SECRET` | `secret` | Secret used to sign session cookies. **Change this in production.** |
    | `LOG_LEVEL` | `info` | Runtime log level: `debug`, `info`, `warn`, or `error`. |
    | `FRONTEND_URL` | `http://localhost:3000` | URL of the frontend, used for generating links in emails and redirects. |
    | `NOTIFICATION_POLL_INTERVAL` | `10000` | Interval in milliseconds at which the frontend polls for new notifications. Set to `0` to disable polling. |
    | `BLOCK_PROTOCOL_API_KEY` | ŌĆö | API key for fetching blocks from the [Block Protocol Hub](https://blockprotocol.org/hub). Generate one at `https://blockprotocol.org/settings/api-keys`. |
  </Accordion>

  <Accordion title="Email">
    HASH supports three email transport mechanisms. In development the `dummy` transporter logs emails to a local file instead of sending them.

    | Variable | Default | Description |
    |----------|---------|-------------|
    | `HASH_EMAIL_TRANSPORTER` | `dummy` | Email transport to use: `dummy`, `aws`, or `smtp`. |
    | `DUMMY_EMAIL_TRANSPORTER_FILE_PATH` | `var/api/dummy-email-transporter/email-dumps.yml` | Path to the file where dummy emails are written in development. |
    | `SYSTEM_EMAIL_SENDER_NAME` | ŌĆö | Display name for outgoing emails. Required when transport is `aws` or `smtp`. |
    | `SYSTEM_EMAIL_ADDRESS` | ŌĆö | From address for outgoing emails. Required when transport is `aws` or `smtp`. |
    | `SMTP_SERVER_HOST` | ŌĆö | Hostname of the SMTP server. Required when transport is `smtp`. |
    | `SMTP_SERVER_PORT` | ŌĆö | Port of the SMTP server. Required when transport is `smtp`. |
    | `SMTP_SERVER_USERNAME` | ŌĆö | SMTP authentication username. Optional if the server auto-authenticates. |
    | `SMTP_SERVER_PASSWORD` | ŌĆö | SMTP authentication password. Optional if the server auto-authenticates. |

    To use the AWS SES transporter, set `HASH_EMAIL_TRANSPORTER=aws` and configure the AWS variables below.
  </Accordion>

  <Accordion title="AWS (S3 uploads and SES email)">
    Used for file uploads (S3) and transactional email (SES). Local file uploads are used by default; S3 is recommended for production.

    | Variable | Default | Description |
    |----------|---------|-------------|
    | `AWS_REGION` | ŌĆö | AWS region, e.g. `us-east-1`. |
    | `AWS_ACCESS_KEY_ID` | ŌĆö | AWS access key ID for general API operations. |
    | `AWS_SECRET_ACCESS_KEY` | ŌĆö | AWS secret access key for general API operations. |
    | `FILE_UPLOAD_PROVIDER` | `LOCAL_FILE_SYSTEM` | Upload provider: `LOCAL_FILE_SYSTEM` or `AWS_S3`. |
    | `LOCAL_FILE_UPLOAD_PATH` | `var/uploads` | Relative path to store files when using the local provider. |
    | `AWS_S3_UPLOADS_BUCKET` | ŌĆö | S3 bucket name for file uploads. Required when `FILE_UPLOAD_PROVIDER=AWS_S3`. |
    | `AWS_S3_UPLOADS_ACCESS_KEY_ID` | ŌĆö | AWS access key ID specifically for S3 uploads. Optional if an IAM role grants access. |
    | `AWS_S3_UPLOADS_SECRET_ACCESS_KEY` | ŌĆö | AWS secret access key specifically for S3 uploads. |
    | `AWS_S3_UPLOADS_ENDPOINT` | ŌĆö | Custom S3-compatible endpoint URL. Useful for MinIO or other providers. |
    | `AWS_S3_UPLOADS_FORCE_PATH_STYLE` | ŌĆö | Set to `true` if your S3 setup requires path-style requests instead of virtual-hosted style. |

    <Note>
      For in-browser document previewing, configure an `Access-Control-Allow-Origin` header on your S3 bucket that is not a wildcard (`*`).
    </Note>
  </Accordion>
