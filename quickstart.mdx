---
title: "Quick Start"
description: "Get HASH up and running in minutes — choose between the hosted cloud service or a local development environment."
---

HASH offers two ways to get started. The cloud option is the fastest path to a working environment. The self-hosted option gives you full control over your data and infrastructure.

<Tabs>
  <Tab title="Cloud (recommended)">
    The quickest way to use HASH is through the hosted service at [app.hash.ai](https://app.hash.ai). No installation required.

    <Steps>
      <Step title="Create an account">
        Go to [app.hash.ai/signup](https://app.hash.ai/signup) and create a free account using your email address.
      </Step>
      <Step title="Sign in">
        After registering, [sign in](https://app.hash.ai/signin) to access your HASH workspace.
      </Step>
      <Step title="Skip the waitlist (if applicable)">
        When you first create an account, you may be placed on a waitlist. To move to the front of the queue, follow the instructions shown in your HASH dashboard after signing in. All submissions are reviewed by a member of the HASH team.
      </Step>
      <Step title="Create your first entity">
        Once inside your workspace:

        1. Click **Entities** in the left sidebar.
        2. Click **New entity**.
        3. Select or create an entity type (for example, `Person` or `Company`).
        4. Fill in the properties for your entity and click **Save**.

        Your entity is now stored in your knowledge graph and can be queried, linked, and enriched by AI agents.
      </Step>
      <Step title="Explore flows and agents">
        Navigate to **Flows** in the sidebar to run an AI-powered research workflow. Flows can automatically gather and structure information from the web into your knowledge graph.
      </Step>
    </Steps>
  </Tab>
  <Tab title="Self-Hosted">
    You can run HASH locally for development or evaluation. This requires installing several dependencies and running Docker containers.

    <Warning>
      Running HASH locally is not yet officially supported. These are experimental instructions. For a reliable experience, use the [hosted cloud service](https://app.hash.ai).
    </Warning>

    <Steps>
      <Step title="Install prerequisites">
        Make sure the following tools are installed and meet the minimum versions:

        - **Git** ≥ 2.17
        - **Rust** (via `rustup`) — version matched to `rust-toolchain.toml`
        - **Docker** ≥ 20.10
        - **Docker Compose** ≥ 2.17.2
        - **Docker Buildx** ≥ 0.10.4
        - **Protobuf** compiler

        On macOS, if `git --version` fails you may need to install Xcode Command Line Tools first:

        ```bash
        xcode-select --install
        ```

        If you use Docker Desktop on macOS or Windows, go to **Preferences → Resources** and allocate at least 4 GB of RAM (8 GB recommended).
      </Step>
      <Step title="Clone the repository">
        ```bash
        git clone https://github.com/hashintel/hash.git
        cd hash
        ```
      </Step>
      <Step title="Install development tools with mise">
        HASH uses [mise-en-place](https://mise.jdx.dev/) to manage consistent tool versions across the codebase:

        ```bash
        mise install
        ```

        After installing mise, make sure it is [activated in your shell](https://mise.jdx.dev/getting-started.html#activate-mise).
      </Step>
      <Step title="Install Node.js dependencies">
        ```bash
        yarn install
        ```
      </Step>
      <Step title="Configure environment variables">
        Create a `.env.local` file in the repository root. Dummy values are sufficient if you are not using AI features:

        ```bash
        OPENAI_API_KEY=your-openai-api-key
        ANTHROPIC_API_KEY=your-anthropic-api-key
        HASH_TEMPORAL_WORKER_AI_AWS_ACCESS_KEY_ID=your-aws-access-key-id
        HASH_TEMPORAL_WORKER_AI_AWS_SECRET_ACCESS_KEY=your-aws-secret-access-key
        E2B_API_KEY=your-e2b-api-key
        ```

        <Tip>
          `.env.local` is not committed to the repository. Put all secrets here. Default values come from `.env` and `.env.development` — override them in `.env.local` without editing those files.
        </Tip>
      </Step>
      <Step title="Start external services">
        Launch Postgres, the Graph query layer, Kratos, and Redis as Docker containers:

        ```bash
        yarn external-services up -d
        ```

        To force a rebuild of the containers (required when the graph query layer has changed):

        ```bash
        yarn external-services up -d --build
        ```
      </Step>
      <Step title="Start the application">
        ```bash
        yarn start
        ```

        This starts both the backend and frontend in a single terminal. Once you see `http://localhost:3000`, the frontend is ready. The API is online once you see `localhost:5001`. Both must be running for the frontend to work.
      </Step>
      <Step title="Sign in with a dev account">
        Three users are seeded automatically for development. All have the password `password`:

        | Email | Role |
        |-------|------|
        | `alice@example.com` | Regular user |
        | `bob@example.com` | Regular user |
        | `admin@example.com` | Admin |

        Open [http://localhost:3000](http://localhost:3000) and sign in with one of these accounts.
      </Step>
    </Steps>

    For full setup details, troubleshooting, and database reset instructions, see the [Self-Hosting HASH](/self-hosting) page.
  </Tab>
</Tabs>
